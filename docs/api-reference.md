# ORBI Phase 1 API Reference

All responses use JSON. Public routes require an ORBI project API key. Administrative routes
require the one-time bootstrap administrator token generated by `scripts/bootstrap-dev.py`.
Worker routes require a short-lived worker JWT.

## Public routes

| Method | Route | Scope | Phase 1 behaviour |
|---|---|---|---|
| `GET` | `/health` | None | Process liveness only |
| `GET` | `/v1/models` | `models:read` | Lists active model aliases |
| `POST` | `/v1/chat/completions` | `chat` | Queues work and waits up to the configured timeout |
| `GET` | `/v1/usage` | `usage:read` | Returns project budgets and key completion totals |

Phase 1 rejects `stream: true`. Server-sent-event streaming is deferred to Phase 2.

## Administrative routes

| Method | Route | Purpose |
|---|---|---|
| `POST` | `/admin/projects` | Create a project tenant |
| `POST` | `/admin/model-aliases` | Register a provider-independent model alias |
| `POST` | `/admin/api-keys` | Create a scoped API key; raw key is returned once |
| `POST` | `/admin/worker-enrollments` | Create a one-time worker enrollment token |
| `POST` | `/admin/api-keys/{id}/revoke` | Revoke a project API key |
| `POST` | `/admin/workers/{id}/revoke` | Revoke a worker JWT and suspend the worker |

## Worker routes

| Method | Route | Required JWT scope |
|---|---|---|
| `POST` | `/internal/workers/register` | One-time enrollment token |
| `POST` | `/internal/workers/heartbeat` | `worker:heartbeat` |
| `GET` | `/internal/workers/request` | `worker:pull` |
| `POST` | `/internal/workers/result` | `worker:result` |
| `POST` | `/internal/workers/error` | `worker:error` |
| `POST` | `/internal/workers/renew` | `worker:renew` |

## Idempotency

Clients should send an `Idempotency-Key` header containing 8–128 URL-safe characters.
PostgreSQL enforces uniqueness for `(api_key_id, idempotency_key)`. Redis is not the correctness
boundary. A completed duplicate returns the previously committed result; an in-progress duplicate
returns its current ORBI request state.

## Error shape

```json
{
  "error": {
    "code": "stable_machine_code",
    "message": "Safe public message"
  }
}
```

Validation responses omit submitted values so prompts and credentials are not reflected back.


## Request limits

The gateway rejects bodies larger than `ORBI_MAX_REQUEST_BODY_BYTES` before parsing JSON. The
default is 1 MiB. Normal responses include `Cache-Control: no-store`, `X-Content-Type-Options:
nosniff`, and an `X-ORBI-Request-ID` correlation header.
